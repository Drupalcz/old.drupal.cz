core.php slouzi pouze jako

Napsal uživatel Marek Sotak dne 21 Červen, 2008 - 22:03.

core.php slouzi pouze jako soubor s dokumentaci. neni soucasti drupalu. veskere funkce, ktere jsou obsazeny v core.php jsou ukazky implementace. jinak co se tyce puvodni definice, tak ta nemusi byt definovana. zalezi jak to udelat ve vlastnim modulu.

Kdyz uz sme u toho volani hooku _help, tak tam je to o neco slozitejsi nez normalne, protoze se nacitaji informace i z .info souboru. Klasicky a jednoduchy priklad bych hledal treba v node.module a jak se vola hook_nodeapi

function node_invoke_nodeapi(&$node, $op, $a3 = NULL, $a4 = NULL) {   $return = array();   foreach (module_implements('nodeapi') as $name) {     $function = $name .'_nodeapi';     $result = $function($node, $op, $a3, $a4);     if (isset($result) && is_array($result)) {       $return = array_merge($return, $result);     }     else if (isset($result)) {       $return[] = $result;     }   }   return $return; }

Tipujem ze v tom core.php

Napsal uživatel Havran dne 21 Červen, 2008 - 22:00.

Tipujem ze v tom core.php su len priklady. hook_funkcia() je len pomenovanie pre hook – ook aj tak vzdy nahradis menom svojho modulu.

Princip je v tom ze aj sam modul ktory vyvola nejaky novy hook cez module_invoke/mo­dule_invoke_a­ll ma nadefinovany ten hook. Ja som to pouzil pre Texy Filter modul kde som nadefinoval hook ktory hlada v ostatnych moduloch funkciu pre nastavenie Texy.

Formatovanie kodu – /--code php na prvom riadku, potom kod a na poslednom riadku \--.
