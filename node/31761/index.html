Změna rozmístění chcek-box buttonů a dalšího (CSS)

Toto je archivovaná verze české komunitní stránky o CMS Drupal. Nejde na ni již přidávat nový obsah. Pro aktuální informace a kontakt s komunitou navštivte Drupal.cz

Kategorie: Ostatní
Týká se verze: Drupal 7.x

Ahoj,

chtěl bych všem poděkovat za péči, kterou se mnou měli při sestavování mého webu v Drupalu. Už jsem se vyhrabal z postavení modulů a parametrů webu a nyní se dostávám do designování.

Při této příležitosti jsem chtěl požádat o radu s rozmístěním různých buttonů apod. Přílohou zasílám 3 obrázky pro názornost:

  1. výchozí aktuální vzhled, který mi moduly a Drupal umožňují
  2. požadovaný vzhled – toto je problémová fáze a také předmět tohoto příspěvku
  3. zdrojový kód pro vás, odborníky, kteří by mě třeba podle něj mohli lépe nasměrovat na konkrétní css prvek.

Takže jak je již asi zřejmé. Jde mi o úpravu rozmístění různých tlačítek NIKOLI pod sebe, jak je nyní výchozí zobrazení, ale VEDLE SEBE, například vždy 4 na řádek (viz. obrázek „pozadovano.png“ v příloze).

Mám zkušenosti s tabulkami pouze v HTML, ale předpokládám, že tady to bude mnohem složitější.

PRO INFO: Tento konkrétní požadavek se týká obsahu vytvořeného pomocí VIEWS + Better Exposed Filters. Napadlo mě také vložit CSS class přímo ve Views do Other v nastavení Advanced. Ale vůbec nevím jak v CSS zajistit rozmístění buttonů jinam, jak rozšířit search field apod. Možná je tedy lepší jít přímo ke zdroji a napevno to upravit tam.

Všechny úpravy (ať už modulů,

Napsal uživatel Petr Illek dne 17 Srpen, 2014 - 07:25.

Všechny úpravy (ať už modulů, javascriptů, nebo tématu vzhledu) nikdy neděláme přímo v jádře nebo v contrib modulu! Při každé aktualizaci byste pak tyto změny musel provádět znovu a to je cesta do pekla (taky jsem si ji před lety při seznamování s Drupalem prošel)

Tuhle změnu proveďte ideálně ve vlastním, nebo odovozeném tématu vzhledu. Jak na odvozování najdete dobře popsané například v tématu ZEN, případně v tomto článku od Evy (https://www.drupal.cz/…tema-vzhledu) Je starší, ale princip je platný. CSS soubory v Drupalu fungují na systému přetěžování. Tzn. později načtené CSS soubory obsahující stejné selectory přepíší dříve načtené CSS soubory. Pořadí CSS souborů je krásně vidět v HTML kódu vaší stránky. Obecně se nejdříve načítají CSS soubory z jádra, následují soubory z modulů a na konci jsou CSS vašeho tématu. Díky tomu máte právě možnost ovlivnit téměř vše.
