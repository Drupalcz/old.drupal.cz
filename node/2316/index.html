Pre rucne zadavanie je

Napsal uživatel Martin Hrabovcin (bez ověření) dne 23 Leden, 2008 - 13:06.

Pre rucne zadavanie je lepsie vyuzit moznost „simulovat“ odoslanie formulara uzivatelom, tak bude zaistene, ze vsetky moduly, ktore sa maju na tvorbe obsahu (node) podielat budu spustene. Pre tento pripad (a nie len pre vytvaranie node ale vseobecne pre formulare) existuje funkcia drupal_execute(for­m_id, values, …)

V tvojom pripade by asi pomohlo nieco take

<?php $form_id = 'ccktype_node_form'; // kde ccktype je typ node  $node = array(); $node['type'] = 'ccktype'; $node['uid'] = 1; // uzivatel, pod ktorym chces vytvorit node  // "zasielane" hodnoty pre node $values['title'] = 'Titulok'; $values['body'] = 'Obsah'; $values['format'] = 1; // format pre body $values['field_myfield'][0]['value'] = 'CCK hodnota pre myfield Pole';  // samotne spustenie formularu drupal_execute($form_id, $values, $node);  // zistenie errorov $errors = form_get_errors(); // ak bude pole > 1 ... zvysok uz je asi jasny  ?>

Jednoduchy tutorial :)

Ja som to uz vyuzival vo

Napsal uživatel Martin Hrabovcin (bez ověření) dne 23 Leden, 2008 - 16:24.

Ja som to uz vyuzival vo viacerych pripadoch. Drupal ma velmi dobre spraveny boot process takze da sa velmi jednoducho pracovat s drupal api ako frameworkom (Form api, database abstraction layer) bez toho aby bolo potrebne pisat moduly, ktore je potrebne vyuzivat iba na jediny pripad.

Mozny priklad je tu.

subor: import.php mozes si nastavit cron pre pravidelnu navstevu

<?php  // tu nastartuje cely drupal include_once './includes/bootstrap.inc'; drupal_bootstrap(DRUPAL_BOOTSTRAP_FULL);  // po tomto momente je k dispozicii cele drupal api // vybudovat $node, $values a potom pustit drupal_execute  // v pripade ze chceme simulovat cinnost urcite uzivatela tak potom global $user; // nie som si isty ci toto treba user_load(array('uid' => 1));  // vsetky kroky dalej budu vykonavane pod adminom so vsetkymi pravami ?>

Co sa tyka tvojho importu tak by som to asi riesil pomocou tohto skriptu a este nejakou triedou by som sa pripojil na msaccess databazu, ktora by ti to pravidelne kontrolovala. Tym by si dosiahol to, ze nebude potrebny vstup od ziadneho uzivatela.

Pozeram ze by z toho celeho mohol byt pekny clanok :)
