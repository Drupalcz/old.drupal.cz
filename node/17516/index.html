Návod ako premigrovať používateľov (userov) z drupal 6 do drupal 7 pomocou modulu Feeds

Toto je archivovaná verze české komunitní stránky o CMS Drupal. Nejde na ni již přidávat nový obsah. Pro aktuální informace a kontakt s komunitou navštivte Drupal.cz
Vložil/a jasom, 8 Únor, 2013 - 12:59

V tomto krátkom návode si ukážeme ako premigrovať používateľov (userov) z drupal 6 do drupal 7 s využitím Calc (Open Office Excell), phpmyadminu a modulu Feeds. Náročnosť – stredne pokročilí (treba vedieť čo je to drupal, myš a klávesnica:-)

Ak chcete updatovať drupal inštaláciu z d6 do d7, najvhodnejšie je začať s migráciou používateľských účtov. Takže začneme:

Krok 1: Export tabuľky USERS do CVS

Ideme do našej d6 databáze (phpmyadmin) do tabuľky users. Až na stĺpec data si ju necháme celú vyexportovať cez SQL dopyt:

SELECT uid , name , pass , mail , mode , sort , threshold , theme , signature , signature_format , created , access , login , status , timezone , language , picture , init FROM users ORDER BY users.uid ASC

Dole klikneme na export a nastavíme si tieto parametre:

  • format: CVS
  • columns separated with: ;
  • zakliknem put columns names in the first row (aby sa v CVSku dalo orientovať)
  • dodatočne, záleží od verzie phpmyadmin, zaklikneme save as file

Súbor, ktorý mi phpmyadmin vypľuje nazveme users_d6.cvs a otvoríme si ho v CALC:

  • Jazyk: Slovenčina (Čeština, Ingliš)
  • Oddelovač: semicolon
  • Formátovanie: UTF-8

Krok 2: Konfigurácia Feeds

Nainštalujte si Feeds a ním požadované moduly Chaos tools a Job Scheduler. Admin interface pre user import (Feeds) je tu.

/admin/structu­re/feeds/user

Odtialto si môžete naimportovať nastavenia, ktoré použijeme my – user_import.txt. V zásade sú to iba factory settings plus:

  • Parser: CSV parser
  • CSV parser: Default delimiter nastav ;
  • Processor: User Processor
  • User processor Mapping:
    • Name of source field musi byť ten, čo je v prvom riadku CVSka pre daný stĺpec:
    • name → User name
    • pass → Unencrypted Password
    • mail → Email address
    • created → Created date
    • status → Account status

Ok, teraz ideme na url webu, kde sa začína importovanie

/import/user

Priložíme súbor a klikneme na import. Začne sa import, ktorý naimportuje userov.

Nemusia sa nám naimportovať všetci používatelia na prvý krát. Feeds nám však spraví log, kde sa dozvieme, pri ktorom účte sa import nepodaril. Ni­ektorí používatelia zadávajú meno alebo heslo s medzerou na konci alebo začiatku, a to Feeds vyhodnotí ako chybu. Vtedy treba riadky v CVS opraviť a naimportovať iba opravené riadky ešte raz. 

Tak a minimálny import userov máme za sebou. Ak sa uspokojíme s týmto stavom, treba ísť ešte do databázy a vymazať stĺpec pass. Heslá sa nám naimportovali zle, pretože v drupal 7 sa heslá už neukladajú v čistom md5 hashe. Inými slovami, používatelia si pred prvým prihlásením po migrácii musia nechať vygenerovať nové heslo. Nič moc a preto pokračujeme ďalej.

Krok 3: Migrácia hesla do drupal 7

Nahradenie d7 hesla za staré, d6 heslo spravíme cez používateľské meno. Do novo pridaného stĺpca sql v u­sers_d6.cvs si spravíme takúto funkciu (mojnazovdata­bazy nezabudni­te nahradiť za váš názov databázy):

=„UPDATE  mojnazovdataba­zy.users SET  pass =  ‘“&C2&„‘ WHERE  users.name =‘“&B2&„‘;“

S takýmto výsledkom:

UPDATE  mojnazovdataba­zy.users SET  pass =  ‘3667cbdafb66c626f2c1d­56c1bd96d69‘ WHERE  users.name =‘Franto‘;

Naklonujeme si vzorec do všetkých riadkov. Celý stĺpec (bez prvého riadku, lebo to je názov stĺpca) si nakopírujeme a pustíme ako SQL príkaz cez phpmyadmin v d7 databáze. Tým sa nám priradí k používateľovi jeho staré MD5 heslo.

V ďalšom kroku yužijeme tento kúsok kódu rehash_php.txt z Stackoverflow­.com. Rehash.php u­miestnime do rootu d7 inštalácie (tam kde máme update.php). Spustíme skript. Ten rehashuje všetky heslá prvým 50000 používateľom s UID > 1.

Hotovo, premigrovali sme aj heslá.

Krok 4: Migrácia dodatočných údajov ako signature, access, login, init, timezone

Ak chceme byť detailisti, a neprísť o nejaké potencionálne cenné údaje, môžeme premigrovať ešte aj stĺpce signature, access, login, init, timezone. Ukážeme si to na príklade access (posledný prístup používateľa):

Pridáme si ďalší stĺpec sql2do u­sers_d6.cvs s ta­kýmto vzorcom:

=„UPDATE  mojnazovdataba­zy.users SET  access =  ‘“&L2&„‘ WHERE  users.name =‘“&B2&„‘;“

A takýmto výsledkom:

UPDATE  mojnazovdataba­zy.users SET  access =  ‘1302533548‘ WHERE  users.name =‘Franto‘;

Tak ako pri heslách si naklonujeme vzorec do všetkých riadkov. Celý stĺpec (bez prvého riadku) si nakopírujeme a pustíme ako SQL príkaz cez phpmyadmin. Obdobne postupujeme aj pre signature, login, init, timezone.

To je všetko. Ak vám niečo nie je jasné, skúste komentáre. Platenú podporu poskytujem cez skype v cene 11,95EUR/hodina. Postup: najprv zaplatiť cez paypal a potom mi napísať tu alebo tu.

Poznámky záverom:

  • teraz si nespomeniem, ako som importoval avatary (treba nazrieť do feed's issu­es), v tomto mojom dnešnom prípade malo avatar iba 7 ľudí, takže som to spravil ručne
  • mám zapnutý iba jeden jazyk, takže jazyky som neriešil
  • users_d6.ods – tu je príklad user_d6 aj so vzorcami

Nabudúce si ukážeme ako naimportovať pomocou Feeds nody, komentáre a termíny.
