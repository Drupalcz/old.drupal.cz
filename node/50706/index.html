Modul Domain - dodefinování dalších konfiguračních údajů k doméně a vyhledávání podle nich

Toto je archivovaná verze české komunitní stránky o CMS Drupal. Nejde na ni již přidávat nový obsah. Pro aktuální informace a kontakt s komunitou navštivte Drupal.cz

Kategorie: Jak na to?
Týká se verze: Drupal 7.x

Ahoj,

už jste mně tu na fóru moc poradili. Nechci aby to vypadalo, že nedokážu hledat :) toto je věc, na kterou jsem opravdu odpověď nenašel.

Mám domény a potřeboval bych k nim dodefinovat další možnost konfigurace. Stačí mně nějaký attribut site_timeout typu datum. Tak budu schopný po založení domény specifikovat, do kdy chci aby daná doména byla aktivní. Jelikož jsem na časově omezéné domény nic hotového nenašel, napsal jsem si vlastní modul. Zatím mně stačí si jen uložit údaj, do kdy je doména aktivní a pak si někde vypsat seznam domén, které už jsou po tomto datu. Nastavení data udělám přes malinký modul:

/**  * Implements hook_menu().  */  function site_variables_menu() {     $items['admin/site-variables'] = array(         'title' => 'Admin variables',         'description' => 'Allows admins to update site variables',         'page callback' => 'drupal_get_form', // this is the function that will be called when the page is accessed.  for a form, use drupal_get_form         'page arguments' => array('site_variables_form'), // put the name of the form here         'access callback' => 'user_access',         'access arguments' => array('access site_variables config'),     );    return $items; }    /**  * Page callback for /site-variables.  */ function site_variables_form($form, &$form_state) {          $form['site_variables']['site_timeout'] = array(                 '#type' => 'date_popup', //you can find a list of available types in the form api                 '#title' => t('Site timeout'),                 '#required' => TRUE, //make this field required                 '#default_value' => variable_get('site_timeout'),         '#date_format' => "Y-m-d",         '#description' => t('Site will be activated only until this time'),         );          $form['submit_button'] = array(                 '#type' => 'submit',                 '#value' => t('Save data')         );          return $form;  } function site_variables_form_submit($form, &$form_state) {     $domain = domain_get_domain();     domain_conf_variable_set($domain['domain_id'], 'site_timeout', $form_state['values']['site_timeout']); }

Toto funguje krásně jak pomásle. Každá doména ma vlastní formulář, kde si můžu specifikovat do kdy má být nastavená. Ještě mě napadlo to zkusit dodefinovat přímo do nastavení domény přes http://api.drupalhelp.net/…in_batch/7.3 ať nastavení nemám na dvou místech.

Nicméně ať tak nebo tak, potřeboval bych potom vypsat domény (nevím zda to jde udělat přes View), které už mají hodnotu nastavenou v proměnné site_timeout < Date.now(). Poraďte prosím nějaký jednoduchý způsob, jak toho dosáhnout aniž bych musel procházet všechny domény a v nich pak hledat. To by nebylo moc efektivní způsob.

Díky moc Martin

Ještě na tom s hookama tak

Napsal uživatel martinpesout dne 3 Červen, 2015 - 21:57.

Ještě na tom s hookama tak dobře nejsem, tak se radši zeptám. K čemu mně je vlastně hook_schema_alter? Měl jsem zato, že by mně mohla zajistit automatické načtení dat do nově přidané formuléřové položky Toto je moje situace nyní:

function site_variables_form_alter(&$form, &$form_state, $form_id) {     if ($form_id != 'domain_form') {         return FALSE;     }      $inserted = array(         'expiration_date' => array(             '#type' => 'date_popup', //you can find a list of available types in the form api             '#title' => t('Domain expiration'),             '#required' => TRUE, //make this field required             '#date_format' => "Y-m-d",             '#description' => t('Domain will be activated only until this time')         )     );      $form = array_slice($form, 0, 5, true) + $inserted + array_slice($form, 5, NULL, true);      $form['#submit'][] = 'site_variables_form_submit'; }    function site_variables_form_submit($form, &$form_state) {     $domain_id = $form['domain_id']['#value'];      $date = $form['expiration_date']['#value']['date'];      db_update('domain')       ->condition('domain_id', $domain_id)       ->fields(array(         'expiration_date' => strtotime($date),       ))       ->execute(); }    /**  * Implementation of hook_schema_alter().  */  function site_variables_schema_alter(&$schema) {   if (isset($schema['domain'])) {     $schema['domain']['fields']['expiration_date'] = array(       'type' => 'int',       'not null' => TRUE,       'default' => 0,       'description' => 'Date of domain expiration.',     );   } }    /**  * Update of database schema  */  function site_variables_update_7370(&$sandbox) {     db_add_field(         'domain',         'expiration_date',         array(             'type' => 'int',             'not null' => TRUE,             'default' => 0,             'description' => 'Date of domain expiration.'         )     ); }

Zdravím. Díky za veškeré rady. Nakonec jsem vše nastudoval a vytvořil modul, který dělá to co potřebuju a navíc upravuje View abych mohl i v přidaném atributu vyhledávat. Měli jste pravdu. S hookama dokážu opradu magické věci :)

site_variables­.install

<?php  /**  * @file  * Install file for the Site Variables module  */  /**  * Implements hook_schema_alter().  */  function site_variables_schema_alter(&$schema) {   if (isset($schema['domain'])) {     $schema['domain']['fields']['expiration_date'] = array(       'type' => 'int',       'null' => TRUE,       'default' => NULL,       'description' => 'Date of domain expiration.'     );   }    return $schema; }    /**  * Implements hook_uninstall().  */  function site_variables_uninstall() {         db_drop_field(                 'domain',                 'expiration_date'         ); }     /**  * Implements hook_install().  */  function site_variables_install() {         db_add_field(         'domain',         'expiration_date',         array(             'type' => 'int',             'null' => TRUE,             'default' => NULL,             'description' => 'Date of domain expiration.'         )     ); }
