Týká se verze:

Ahoj, mám takovou základní vizi, kterou bych rád realizoval v Drupalu. Nicméně nějak se nedokážu rozhodnout, jak udělat první krok.

Velmi stručně: Jde mi o to, vytvořit určitý katalog (lidí). K tomuto účelu mám stvořené CSV s detailními informacemi, (včetně zeměpis. délky, šířky pro vytvoření mapového „mashupu“:) Nevím však, zda pro tento speciální typ obsahu vytvořit pomocí CCK, nebo obohatit o tyto spec. položky Uživalský profil a provést následně user import, nebo jak na to?? 

Zároveň bych těmto lidem chtel nabídnout moznost plnit obsah (klasika – forum, blogy, clanky,..) i svůj profil – tedy možnost přihlásit se jako user (k předdefinovanému účtu?), atd.

Co mě je ještě dosti trápí jako problém je to, že v seznamu CSV nemám k těmto lidem email. Jak na importu uživatelů bez emailu jsem se zkoušel ptát http://www.drupal.cz/…ilove-adresy.

Můžete mi někdo pomoci a nakopnout?

Díky moc.

V navode k node_profile je

Napsal uživatel Havran dne 3 Leden, 2008 - 17:55.

V navode k node_profile je napisane co s tym. Vyzaduje to tusim este jednen modul. V skratke – cez CCK si vytvoris novy typ obsahu, napriklad profil a nastavis ho ako typ obsahu ktory moze vytvorit kazdy uzivatel len jeden (alebo teda sa mu vytvori prazdny automaticky) ← toto v spojeni s tym dalsim modulom (dohladane → konkretne Node profile + Usernode). V type obsahu si nadefinujes vsetky polia ktore potrebujes v profile pouzivatela (aby boli vyplnene).

Zobrazenie profilu v nejakom peknom tvare je o vytvoreni a modifikovani sablony node-profil.tpl.php v tvojej teme. Atd. je toho viac – urcite si pohladaj na Drupal.org – alebo na internete – bol na jednom blogu navod ako vytvorit a za pomoci akych modulov osobitny profil pre registrovaneho pouzivatela.

User import, Node import, Node profile, User node, ...

Napsal uživatel mat. dne 8 Leden, 2008 - 15:19.

Tvůj postup jsem prozkousel… nicmene jsem se nikam neposunul. Nyni vsak problem mozna zdefinuji lépe :-)

Mám tabulku lidí, ze kterých chci udelat uzivatele. V tabulce mam nejen jmeno, prijmeni, adresu, ale take longitude a latitude a dalsi (rad bych nabidl neco na styl mashupu..) (problém, že nemám jejich emaily, je zásadní, ale řešitelný..)

A ) K rozšíření profilu a požadovaných info můžu využít modul Profile. K automatickemu naimportovani mohu vyuzit modul User Import… Bohuzel tento neumi zpristupnit jiné Drupal fields než jen ty vytvořené modulem Profile, tedy ne adresní položky jako např. Longitude a Latitude modulu Location (resp. GMaps Location)!! :-( Cimz dostanu hromadu lidi coby uzivatele, ale bez vsech atributu, ktere bych rad (hlavně délky a šířky:)… Takže není to ideální řešení.

Pozn. Kdyby modul User Import uměl zpřístupnit položky Location od usera, asi by to pro mě vše vyřešilo…

B ) Nebo muzu vytvorit pomocí CCK vlastní typ (jak pises Havrane, napr. „profile“) a zde nastavim vsechno mozne, vč. povolení Location pro tento typ obsahu a tím zpřístupnění lat. a longt.. Poté mohu využít modul Node import, který mi moje CSV dokáže naimportovat včetně longt a latt. Tím mi ale vzniknou standardní „nody“, které toho moc neumí (a já potřebuji, aby toho uměly jako „user“).

C ) Nabízí se modul Node profile (+Usernode), kdy můžu nastavit, že bude platit 1 uživatel má pouze 1 svůj „nód“ (node profil). Ale co to řeší??

Lze nějak zkombinovat A, B, C tak, abych využil všechny data z mé tabulky (včetně long a lat, abych je mohl zobrazit na mapě) ?

Základní vizí je to, že návštěvník stránek uvidí svůj profil, který si bude chtít editovat a doplnit a dále pak využít všechny možnosti i jinak plnit web (blog, články, atd…). Treba na to jdu uplne blbe :-)

Takze naznacim ako by som to

Napsal uživatel Havran dne 8 Leden, 2008 - 23:14.

Takze naznacim ako by som to riesil ja. Importol by som hromadne pouzivatelov (pravdepodobne by som im hned priradil nejaku rolu) a zistil by som si ktori to su. Vytiahol by som si ich z databazy a ich loginy by som pouzil ako spojovnik k udajom a udaje by som doplnil o uid pouzivatela. A potom by som ich naimportoval do nodeprofile s tym ze by som podla uid priradoval profily.

Znie to jednoducho ale zalezi od toho ako funguju tie importy.
