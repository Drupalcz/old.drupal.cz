Jak na úvodník

Toto je archivovaná verze české komunitní stránky o CMS Drupal. Nejde na ni již přidávat nový obsah. Pro aktuální informace a kontakt s komunitou navštivte Drupal.cz
Vložil/a anonym, 3 Září, 2007 - 07:10

Drupal je jistě perfektní systém pro mnoho různých situací použití. Jenže právě kvůli tomu je to spíš kus plastelíny, kterou každý dostane v papírové krabičce a vymodelovat si z něj toho konkrétního správce obsahu je již na každém jedinci samotném. Jedním z častých použití jsou i informační weby, konkrétněji internetové magazíny, kde ale narážíme na častý problém poněkud nevhodně zvládnuté funkce úvodníku. Kdy je navíc toto z pohledu většinového uživatele používán suplující prvek <!- -break-->. Z pohledu mého je to velmi nedostačující řešení, protože vlastně žádný teaser netvoří, ale pouze ve výpisu článků zobrazí text nad tímto tagem a v samotném článku jej ohraničí pomocí <p></p> jako kterýkoliv jiný odstavec. Nehledě, že při provozu právě nějakého zmíněného internetového časopisu, kde je zcela normální přispívání do těchto periodik více autory, kteří samozřejmě mají přístup do administrace (proč taky ne, když to Drupal v rovině práv dokáže zpřístupnit), je toto nepoužitelné, protože se tak jistě bude často stávat, že prostě nějaký autor na toto zapomene.

Toto jsem v nedávné době také řešil a zde v diskuzi mi bylo porazeno použit modul Content Templates (Contemplate) spolu ve dvojici s modulem Content Construction Kit (CCK). Pro umocnění jakéhosi návodového stylu budu následující text prokládat obrázky pro lepší demonstraci a snad i četnost samotného textu.

Stáhnutí a instalace

Z oficiálních stránek Drupalu si stáhněte moduly CCK a Kontemplace a obyčejně jako každé moduly je instalujte překopírováním do složky /modules a následně je aktivujte v administraci.

Poté v administračním rozhraní (Domů › Administer › Content management) upravte obsahový typ Story tak, že mu přidáte nové pole, které nazvete třeba uvodnik nebo perex, není to vůbec důležité, má to jen orientační funkci.

Nastavíte mu skrytí popisu a výhu (prioritu, umístění) tak, že bude před obsahem (body), ale až za nadpisem (title). Také můžete zvýšit počet řádků, aby v administraci získal větší okno, ne pouze jeden řádek jako titulek.

Nyní se přesuť v administraci (admin/conten­t/templates) a otevřete si položku editace šablony článku, kde následně odškrtnutím povolte změny pro úvodník (teaser) a článek (body). Pozor, nejedná se o části článku, tedy zobrazení stránky Story, to je zde vedeno jako „body“, ale o jednotlivé součásti. Tedy „teaser“ je box s nadpisem a něčím pod ním na zobrazení výpisu článků.

Do obou napište něco ve smyslu obrázků, což zapříčiní, že se ve výpisu článků zobrazí jen pole „uvodnik“ a v samotném otevřeném článku pak jak perex, tak článek. Ovšem pomocí divů toto půjde nastylovat, na každé stránce jinak, samozřejmě.

Díky tabulky vedle je samozřejmě možno rychle a velmi jednoduše změnit chování všech prvků (teaser, body, rss) a jelikož se v ní zobrazují i vlivy podle posledního článku, které to bude mít. Je nastavení velice intuitivní.

Toto řešení mi příjde o hodně lepší než moduly Excerpt a Nodeteaser.

katalog je CCK + Content

Napsal uživatel Jakub Suchý dne 3 Září, 2007 - 22:52.

katalog je CCK + Content Templates + Views + jeden vlastni modul ktery vypada takhle:

function katalog_display() {   $view = views_get_view(3);   if (!$view) { drupal_not_found(); exit; }    $term = arg(2);   if ($term) {     // musime najit term, ktery je pro tenhle view     $res = db_query("select * from term_data where vid = 10 OR vid = 9");     while ($row = db_fetch_array($res)) {       $cleanterm = pathauto_cleanstring($row['name']);       if ($term === $cleanterm) {         $tid = $row['tid'];         break;       }     }     if (!$tid) {       drupal_not_found(); exit;     }   }    $content = '<p class="katalog-text">Toto je katalog českých webů postavených na Drupalu. Katalog je členěn do kategorií, nebo-li tagů, každý záznam může být v několika kategoriích. Přesvěčte se sami, že Drupal není jen hračka, ale seriózní systém! <a href="/node/add/content-katalog/index.html">Přidat záznam do katalogu</a>.</p>';    // pokud neni tid, zobrazime jen tagy   if (!arg(1)|| arg(2)) {     $tabulka = views_build_view('embed', $view, array(0 => $tid), $view->use_pager, 30);   }    $tags = tagadelic_block('view', '10');   $tagadelic = '<div class="katalog-tags">'.$tags['content'].'</div>';    return $content.$tagadelic.$tabulka; }

jeho ucelem je spojit blok tagadelic s views (coz by asi dneska slo pomoci Views Fusion) a cely to spojit s tema tagama
