Podle mě

Napsal uživatel Eva Rázgová dne 28 Leden, 2014 - 07:01.

Podle mě https://drupal.org/…eference_url nebo https://drupal.org/…_prepopulate (podle toho, jestli používáte entity reference nebo node reference), ale není ani velký problém udělat to ručně.

Princip je, že odkaz musí posílat nějaký argument do URL a ten pak použijete pro předvyplnění pole „odkud uživatel přišel“.

Moje původní myšlenka byla,

Napsal uživatel Eva Rázgová dne 29 Leden, 2014 - 10:16.

Moje původní myšlenka byla, že by se jednalo o 2 klasické uzly (2 typy obsahu), propojené přes Node Reference, + https://drupal.org/…eference_url (a ten modul už si argument i odkaz sám zařídí – dělá se to konfiguračně).

Resp. pak vidíte, jak ten argument vypadá, a pokud by Vám nevyhovovala podoba linku přidaného modulem, můžete si vytvořit link vlastní (a argument musí vypadat stejně).

Jestli se toto dá použít i spolu s webformem, popravdě nevím. Ale viděl byste tam ten princip, což podle mě pomůže.

Jak psal kolega „Anonym“, to může být pro webform možná lepší, ale nedokážu posoudit – pro webform jsem to asi nikdy nepotřebovala.

Odesílání mailu z content

Napsal uživatel Petr Illek dne 30 Leden, 2014 - 08:36.

Odesílání mailu z content type určitě lépe přes modul Rules (https://drupal.org/project/rules). Případně ještě v kombinaci s modulem Message (https://drupal.org/project/message) & Message Notify (https://drupal.org/…ssage_notify).
