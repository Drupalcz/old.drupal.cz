(Například, ale nejen, po aktivaci modulu color a nastavení barev tématu).

Jděte do adresáře, kam se nahrávají soubory (většinou sites/default/fi­les) a najděte soubor .htaccess. Zakomentujte v něm všechny řádky (= na začátek každé řádky napište #) a uložte.

Pozn. – při instalaci některých dalších modulů si Drupal vytváří soubor .htaccess v dalších podadresářích. Při případných problémech s „nezobrazením něčeho“ se podívejte, jestli někde není nový soubor .htaccess, a vyzkoušejte totéž řešení.

Dále zkontrolujte v administraci Nastavení webu > Systém souborů (a případné chybové hlášky).

Používáte-li modul ImageCache, zkontrolujte, zda máte zapnuto ImageAPI GD2.

Pokud se jedná o obrázky vkládané přes FCKeditor nebo CKeditor (event. + IMCE), zkontrolujte Nastavení webu > Vstupní formáty. Máte tag pro obrázky povolený ve Filtered HTML? (Popřípadě, můžete pro pole s obrázky používat Full HTML? Bezpečně lze použít pouze pro prověřené uživatele typu redaktor, editor…). Dále viz README soubor modulu FCKeditor nebo CKeditor.

Zkontrolujte cesty k obrázkům. Jestliže jsou v názvech nebo adresářích obrázků čárky, háčky nebo mezery, je dost pravděpodobné, že je to příčina problému.

EDIT: jestliže se problém týká IE, je to pravděpodobně tím, že IE zatím neumí pracovat s více než 31 CSS soubory. Problém odstraníte tak, že zapnete komprimaci CSS v Drupalu (na stránce nastavení výkonu). Pro vývoj použijte modul pro částečnou komprimaci CSS: http://drupal.org/…ss_optimizer
