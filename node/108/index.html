Příliš mnoho komentářů?

Toto je archivovaná verze české komunitní stránky o CMS Drupal. Nejde na ni již přidávat nový obsah. Pro aktuální informace a kontakt s komunitou navštivte Drupal.cz
Vložil/a Brumisek, 29 Listopad, 2006 - 23:00

Nedávno jsem se potýkal s problémem, jak zpřístupnit uživatelům portáluMoje Rodina delší historii komentářů. Jak víte, jedna z nejsledovanějších částí portálů a e-magazínů jsou právě poslední komentáře uživatelů. Lidé se totiž chtějí bavit a pokud je uživatelů hodně, často „utečou“ zprávy do nenávratna.

V drupalu máme na komentáře základní blok s názvem Poslední komentáře. Developer tohoto modulu však nedal administrátorům možnost měnit jeho délku – standardně je nastaven na 10 posledních zápisů. Většinou je možné editovat určité parametry bloků na cestě …/admin/block/con­figure/nazev_blo­ku, ale zde na tuto možnost autor nemyslel. Pojďme se tedy podívat do zdrojového souboru comment.module a najděme si, kde se dálka komentáře nastavuje. Vyhledejte funkci function theme_comment_bloc­k() a v ní tento řádek:

$result = db_query_range(db_rewrite_sql('SELECT c.nid, c.* FROM {comments} c INNER JOIN {node} n ON n.nid = c.nid WHERE n.status = 1 AND c.status = %d ORDER BY c.timestamp DESC', 'c'), COMMENT_PUBLISHED, 0, 10);

Jak vidíte, délka seznamu je definována pomocí LIMITního požadavku do databáze. (0,10). Pokud změníme tuto hodnotu, bude náš seznam delší. Například takto:

$result = db_query_range(db_rewrite_sql('SELECT c.nid, c.* FROM {comments} c INNER JOIN {node} n ON n.nid = c.nid WHERE n.status = 1 AND c.status = %d ORDER BY c.timestamp DESC', 'c'), COMMENT_PUBLISHED, 0, 20);

Poznámka redakce: do kódu samotných modulů se snažte zasahovat co nejméně. Ušetříte si práci s pozdějším upgradem. Přesto je tato informace užitečná, proto ji zařazujeme…

Dobrá, délku bloku máme upravnou, ale přecijenom – co kdybychom chtěli zobrazit posledních 100 komentářů? V takovém případě bude blok velmi dlouhý a stránka se bude navíc mnohem déle načítat (pokud je blok posledních komentářů na každé straně, zbytečně byste tak zatěžoval váš server). Šla by vytvořit stránka s posledními komentáři? Ano, máme tu sice tracker, kde můžeme sledovat všechny možné příspěvky za poslední dobu, ale jednoduché zobrazení posledních komentářů tu není.

Trochu jsem se s tím natrápil, ale nakonec jsem vytvořil velmi jednoduchý, ale účelný snippet v PHP, který zobrazuje poslední komentáře v libovolné délce. Tento snippet zobrazí v libovolném počtu zápisů nadpis, autora, uplynulý čas, ze samotného textu příspěvku odstraní HTML značky (pro předejití případných problémů s rozdělením tagů) a upraví jeho délku. Stačí tedy, když vytvoříte novou stránku ve formátu php a vložíte tento kód:

<?php $max = 200; // počet zobrazovaných záznamů $comment_lenght = 140; // maximální délka textu $query = 'SELECT nid,cid,subject,name, timestamp,comment FROM {comments} ORDER BY timestamp DESC'; $result = db_query_range($query, 0, $max);     $items = array();     while ($comment = db_fetch_object($result)) {        $time =  preg_replace("/ /","",format_interval(time() - $comment->timestamp));        if (strlen(strip_tags($comment->comment)) > $comment_lenght) {           $more = '... '; // pokud je text delší, vytisknou se tři tečky       } else {           $more = ' ';       } $items[] ='<b>'.l(check_markup($comment->subject), 'node/'. $comment->nid, NULL, NULL, 'comment-'. $comment->cid).'</b> <small>/ '.check_markup($comment->name).', před '.format_interval(time() - $comment->timestamp).'</small><br />'.substr(strip_tags($comment->comment),0,$comment_lenght).$more.'<br />';     }   $output = "<div class=\"latest-comment-list\">";    if (isset($items)) {     foreach ($items as $item) {       $output .= '<span class="komentar">'. $item .'</span><br/>';     }   }    $output .= '</div>';   print $output; ?>

Pokud vás zajímá jak vypadá výsledek, klikněte zde. Tuto stránku nezapomeňte přidat do menu, tak aby se k ní dostali vaši čtenáři!
