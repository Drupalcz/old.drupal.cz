Jak napsat vlastní modul pro Drupal - díl 2.

Toto je archivovaná verze české komunitní stránky o CMS Drupal. Nejde na ni již přidávat nový obsah. Pro aktuální informace a kontakt s komunitou navštivte Drupal.cz
Vložil/a djanku, 21 Srpen, 2007 - 10:10

Vítejte v dalším díle seriálu, věnovanému vytváření modulů pro Drupal. Dnes se podíváme na to, jak zabezpečit jednotlivé části modulu a také na to, jak obsloužit akce s uživatelským úč­tem.

Veškerý uveřejněný kód navazuje na testovací modul testmod, který jsme vytvořili v 1. dílu.

Hook funkce

O hook funkcích jsem se zběžně zmínil již v prvním díle. Proto neuškodí, když si ony informace zopakujeme nyní, když je budeme prakticky využívat. Hook funkce jsou klasické PHP funkce (nejen), kde se nahrazuje obecná část názvu konkrétním jménem. Tato praktika se využívá zejména v plug-in modelech a drupalovské moduly nejsou vyjímkou. Aplikace, která moduly obsluhuje, pak zjístí, které funkce jsou k dispozici a podle toho je volá. Tento postup zabezpečuje situaci, kdy by mohla být volána funkce, která není implementována.

Všechny hook funkce, dostupné pro moduly, můžete shlédnout na této URL.

Zabezpečení a menu modulu

Drupal nám nabízí rozhraní, pomocí kterého můžeme určit, které části modulu chceme zpřístupnit uživateli a které ne. První hook funkci, kterou si v této souvislosti ukážeme je hook_perm(). Tato funkce vrací pole řetězců, které reprezetují jednotlivá oprávnění. Ty pak můžete přiřadit jednotlivým skupinám.

function testmod_perm() {   return array('Testmod user access', 'Testmod admin access'); }

Oprávněnost pak můžeme otestovat funkcí user_access()­. Např.:

if (user_access('Testmod user access')) echo 'Přístup povolen.';   else drupal_access_denied();

Další cesta vede přes definici položek menu. Toto menu vrací funkce hook_menu(). Jedná se opět o pole, ve kterém jsou definovány jednotlivé položky, včetně cesty, typu, nadpisu a oprávnění. Rozbor jednotlivých typů položek nezapadá do tématu vytváření modulů, proto případné zájemce odkazuji zde.

Následující kód je součástí modulu testmod. Nedělá nic jiného, než, že přesměrovává jednotlivé url adresy na funkci v našem modulu. Tím, že jsme ještě přidali kontrolu oprávnění, jsme zabezpečili i části jen pro danou skupinu uživatelů.

function testmod_menu() {   $polozky = array();    // Vytvoření skupiny položek v administrační části   $polozky[] = array(     'path' => 'admin/testmod',     'title' => t('Testmod administration'),     'description' => t("Administration tools for testmod module."),     'position' => 'left',     'callback' => 'system_admin_menu_block_page',     'access' => user_access('Testmod admin access'),   );    // Položka v administační části   $polozky[] = array(     'path' => 'admin/testmod/polozka',     'title' => t('Testmod administration item :)'),     'callback' => 'administrace',     'weight' => 0,     'access' => user_access('Testmod admin access'),     'type' => MENU_NORMAL_ITEM,    );    // 2 callbacky pro presmerovani adresy na funkci stranky   $polozky[] = array(     'path' => 'testmod/prvni',     'callback' => 'stranky',     'callback arguments' => 'prvni',     'access' => user_access('Testmod user access'),     'type' => MENU_CALLBACK,   );    $polozky[] = array(     'path' => 'testmod/druha',     'callback' => 'stranky',     'callback arguments' => 'druha',     'access' => user_access('Testmod user access'),     'type' => MENU_CALLBACK,   );    return $polozky; }  function administrace() {   $obsah = t("Welcome into administration section!");    print theme('page', $obsah); }   function stranky($stranka) {    $obsah = '';    // Nastavíme titulek   drupal_set_title(t('Page title'));    // Stranku muzeme ziskat pomoci parametru funkce,   // nebo pomoci funkce arg(cislo)    switch ($stranka) {     case 'prvni':       $obsah .= "Toto je první stránka testovacího modulu. <br/>";       $obsah .= l("Odkaz na druhou stránku", "testmod/druha");       break;      case 'druha':       $obsah .= "Toto je druhá stránka testovacího modulu. <br/>";       $obsah .= l("Odkaz na první stránku", "testmod/prvni");       break;   }    print theme('page', $obsah); }

Obsluha operací s uživatelskými účty

Dřív nebo později budeme potřebovat zareagovat na operace s účty uživatelů. Drupal nám k tomu nabízí funkci hook_user(), která je volána pokaždé, když k nějaké operaci dojde. Je na programátorovi modulu, kterých událostí si bude všímat a kterých ne. Já uvedu příklad jen se 4 nejběžnějšími: přihlášení, odhlášení, založení a smazání účtu. Možných událostí je však daleko více. Můžete například pracovat s formulářem, týkající se informací o uživateli nebo přidávat data do informací o uživatelském účtu atp. Kompletní popis najdete na této adrese.

function testmod_user($operace, &$edit, &$user) {    $id = $user->uid;      // ID uctu   $email = $user->mail;  // Email uzivatele   $nazev = $user->name;  // nazev uctu    // V proměnné edit se předávají hodnoty formulářů    switch($operace) {     case 'login':      drupal_set_message("Uzivatel $nazev ($email) byl prihlasen.");      break;      case 'logout/index.html':      // odhlaseni      break;      case 'insert':      // Vytvoreni noveho uctu      // muzeme zde napriklad umistit kod, ktery zapise nova data do databaze      break;      case 'delete':      // Vymaz uctu      break;   }  }

Periodické vykonávaní kódu

Časem možná narazíte na potřebu vykonávat svůj kód v určitých intervalech. Tuto operaci většinou zajišťuje démon (na UNIXových systémech cron), který ve vámi zvolených intervalech spustí skript cron.php v kořenovém adresáři drupalovského webu. Ten rovněž zajistí volání funkce hook_cron(), je-li dostupná.

function testmod_cron() {   // zde umístite vykonávaný kód }

Využití plánovaného vykonávání skriptu se hodí například u posílání (vyžádaných ;) emailů, novinek a nebo také pro pročištění disku atp.

IMHO je hook_menu jeden

Napsal uživatel Havran dne 21 Srpen, 2007 - 21:34.

IMHO je hook_menu jeden z najdolezitejsich prvkov pre tvorbu vlastneho modulu. Jeho pochopenie a princip tvorby menu v Drupale vlastne podla mna tvori zaklad na budovanie kazdeho vlastneho modulu.

Napriklad MENU_CALLBACK, alebo LOCAL_TASK a DEFAUL_LOCAL TASK v menu su velmi dolezite pre tvorbu administracie. Tiez je dobre mat zistene kedy je dolezite tvorit pole $item[] v casti if ($may_cache) { ... } a kedy ich dat do else { } (to tam vobec nemas – chyba ti v hook_menu parameter $may_cache). Pokial je menu v prvej casti podmienky, tak to znamena ze je jeho obsah kesovatelny, nemusi sa vykonat vzdy – v druhej casti naopak nie je a vykona sa vzdy. To je napriklad vhodne na to aby clovek v tejto casti vkladal vlastne CSS styly pre svoj modul…
