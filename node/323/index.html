Vícejazyčná webovka?

Toto je archivovaná verze české komunitní stránky o CMS Drupal. Nejde na ni již přidávat nový obsah. Pro aktuální informace a kontakt s komunitou navštivte Drupal.cz
Vložil/a Šnek, 7 Listopad, 2006 - 11:38
Kategorie: Jak na to?
Týká se verze:
Potřebuji vytvořit webové stránky se stromovou strukturou, u níž by každá stránka měla jednu až pět jazykových verzí, vzájemně přímo přepínatelných (tedy zařazených na stejné místo do stromové struktury). Jde to v Drupalu nějak udělat?

S modulem i18n se mi nepodařilo vůbec definovat různojazyčné verze týchž stránek. Našel jsem modul localizer, který to zvládá, ale jen částečně (anonym nemá přístup na jinojazyčné stránky, a nejde to pravděpodobně nijak přenastavit, a překlad se sice uloží, ale zároveň se na přeložené stránce zaznamená jako vložení nového článku).

Máte někdo zkušenosti s vícejazyčnými webovkami? Jde něco takového, jako potřebuji, v Drupalu vůbec udělat?

Různé jazykové varianty

Napsal uživatel LR (bez ověření) dne 7 Listopad, 2006 - 14:02.

V zásadě je všechno podrobně popsáno v návodu k i18n. Vytváření vazeb mezi kategoriemi je trochu krkolomné, ale jde to. Definuješ kategorii ve všech jazycích zvlášť a pak je ještě musíš propojit. Jazykové mutace článku můžeš přidat ze zobrazení článku – je tam odkaz na překlad.

Koukni se na http://www.soumar.cz, tam je používán modul i18n a Drupal 4.6.

Jenže to je něco jiného, než potřebuji

Napsal uživatel Šnek dne 7 Listopad, 2006 - 15:00.

Našel jsem návod, jak prohlásit zvlášť vytvořený uzel za jinou jazykovou variantu předchozího uzlu, ale nenašel jsem žádné udělátko na plynulé přepínání. Teoreticky by to šlo naprogramovat v PHP. Já ovšem potřebuji CMS, který bude různojazyčné verze takříkajíc genericky podporovat, abych nemusel všem spolupracovníkům vysvětlovat, jak vkládat různé komické kódy. Potřeboval bych něco, kde bych nejprve nadefinoval možné jazyky, a pak měl u každého uzlu možnost editace jiných jazykových verzí, a přepínání se tomu automaticky přizpůsobilo. Pokud tohle v Drupalu nejde, pak prostě Drupal není CMS, který hledám.

Jo, a mimochodem, http://www.soumar.cz/ mi zobrazuje jen prázdnou stránku.

Prepinani obsahu se da

Napsal uživatel Jakub Suchý dne 7 Listopad, 2006 - 16:07.

Prepinani obsahu se da udelat pomoci nekolika bloku a zaroven pod kazdym node by mel byt odkaz na vsechny jeho preklady (coz jsem ja treba vypnul). Mam napsany vlastni blok, ktery udela blok a odkaze primo na preklad v zavislosti na jazyku. Myslim, ze to jde resit lepe, ale ja to mam takhle.

<?php function langblock_help($section='') {   $output = '';   switch ($section) {         case "admin/modules#description":       $output = t("Display language block depending on i18n settings");       break;   }   return $output; }  function langblock_perm() { return array('access content'); }  function langblock_block($op = 'list', $delta = 0) {   if ($op == 'list') {     $blocks[0]['info'] = t('My Languages');   }   elseif($op == 'view') {     $blocks['subject'] = t('My Languages');     $blocks['content'] = theme('item_list', langblock_get_links(1, 0, ''));   }    return $blocks; }  function langblock_get_links($flags = 1, $names = 1, $separator = '') {   $i18n_lang = i18n_get_lang();   $languages = i18n_supported_languages();   // get node link   $nl = substr(i18n_get_normal_path(_i18n_get_original_path()), strlen('node/'));   foreach ($languages as $lang => $langname) {     $name = $names ? $langname: '' ; // Should be localized??     $flag= $flags ? i18n_flag($lang, array('alt'=>$langname)) : '' ;     // get actual path to translated content     $link = i18n_node_l($nl, $lang);     $url = $link ? 'node/'.$link : $lang;     $links[]= i18n_l("$flag$separator$name", $url);   }   return $links; } ?>

Tohle dej do adresare moduls s nazvem langblock.module. V admin → modules ho pak aktivuj a pak aktivuj blok My Languages. Myslim, ze to chodi, ale nejsem si jist :)

jdi na admin → settings

Napsal uživatel Jakub Suchý dne 7 Listopad, 2006 - 16:03.

jdi na admin → settings → i18n. Zde zaskrtni jednotlive typy, ktere chces prekladat v casti Multilingual content. Pak nastav Content selection mode na „Only current language“ – to zajisti, ze se nebudou zobrazovat vsechny preklady najednou.

Pak vytvor node, hned pri vytvareni by se te to melo zeptat na jeho jazyk. Pote jak mas nad kazdym node tab „zobrazit“, „editovat“, mel bys tam mit novy tab „translate“

Teď bych potřeboval dořešit ještě dvě drobnosti:

(2) Některé stránky budou mít překlad do čtyř nebo více jazyků, jiné budou jednojazyčné nebo ve dvou jazycích. U těch jednojazyčných mohu úplně vypnout jazykové přepínátko, ale třeba u dvojjazyčných bych rád nevypisoval celé menu jazyků, ale jen ty existující.
