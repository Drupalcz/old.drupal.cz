Napsal uživatel Radek Pliska dne 11 Únor, 2007 - 21:43.

Díky všem za návod a pomoc. Dobrá věc se povedla. Odpověď čím to může být způsobeno od autora modulu je tady: http://drupal.org/node/113761 To pomohlo aby se nevytvářely další alba. Vyskytla se další chyba. Nemohl sem se dostat do kořenového alba. Ptal sem se autora. Tady je odpověď: It sounds like Acidfree is confused what your root album is… And since you told it not to fix it, it won't. :)

You need to determine what album is the root album. You should be able to do this by doing the query:

mysql> SELECT child FROM acidfree_hierarchy WHERE parent=‚-1‘;

If the query gives you a single row, things are looking good for you. But then if it was giving you a single row, you probably wouldn't be having problems.

if that query shows up empty, you need to create a root album… I am not sure you can do that manually. I suppose you could always try. Just follow the usual links to create an album. After it create the album, go into the database and run the following sql:

mysql> UPDATE acidfree_hierarchy SET parent=‚-1‘ WHERE child=xyz;

where xyz is the nid of the album you just created. Then things should be just fine again.

Diky tomu jsem zjistil že mam několik alb s paret=-1. Odstranil sem ostatní, nechal jeden a chyba je odstraněna.

Kdyby někdo potřeboval s podobným problémem pomoct budu to mít uložené.

Ještě jednou diky. Radek
