ako nezobrazovat taxonomy select box (mam content typu pre presne jeden termin zo slovnika)

Toto je archivovaná verze české komunitní stránky o CMS Drupal. Nejde na ni již přidávat nový obsah. Pro aktuální informace a kontakt s komunitou navštivte Drupal.cz

Kategorie: Jak na to?
Týká se verze: Drupal 5.x

ahojte

mam dotaz na taxonomy
napr mam content type blog ktory ma slovnik „blog“ a terminy („kultura“, „sport“, „zabava“)

to je ok, vsetko jasne.
ako ale na content type ktory je jednoznacny, napr. „hladam spolujazdu“ a ma zmysel pre presne jeden termin?
mozem ho zaradit do slovnika spolujazda, ktory bude obsahovat terminy „hladam spolujazdu“ a „ponukam spolujazdu“.
Potom ale pri vytvarani „hladania spolujazdy“ bude musiet autor vzdy NELOGICKY vyberat v taxonomy selectboxe „spolujazda“ medzi terminami „hladam spolujazdu“ a „ponukam spolujazdu“.

a pritom pre tento content type ma zmysel iba termin „hladam spolujazdu“.
Trocha to mozem zjednodusit ked pouzijem modul taxonomy default, v tom pripade v tom selectboxe bude hned „hladam spolujazdu“, no ale mne ide o princip, naco tam ten select box je, pokial nema vyznam? ked na neho uzivatel klikne tak sa mu zobrazia options hladam spolujazdu a ponukam spolujazdu, co pre neho nema vyznam, kedze je na content type „hladam spolujazdu“

da sa pridelit content typu nejaky taxonomy term „na pozadi“? proste zadefinovat ze tento content type patri do tohoto terminu v tomto slovniku, a na stranke sa to vobec neobjavi.

mozno sa pytam na jednoduchu vec, ale taxonomy niekedy proste nechapem :)

diky vsetkym za pomoc!
 Igorik

standard

Napsal uživatel igorik dne 17 Duben, 2008 - 15:22.

no, akceptujem tvoj nazor, ale podla mna to co chcem nie je nic nestandardne.

vlastny slovnik pre kazdy content type nechcem, lebo vtedy uz taxonomy uplne strati vyznam.
takto su pod jednym slovnikom zoradene logicky podobne terminy.
napr:
spolujazda: (ponukam spolujazdu, hladam spolujazdu),
alebo
Ubytovanie: (ponukam izbu,. hladam izbu, ponukam byt, hladam byt)

dva typy obsahu mam, pisem to v tom clanku vsade, ze ponuka spolujazdy je jeden content type s vlastnymi CCK fieldami, a hladanie spolujazdy je iny content type s inymi CCK fieldami.

vyznam taxonomy je napr. v tom, ked chcem dat uzivatelovi moznost sledovat jednotlive terminy a ked sa objavi nieco nove, tak mu pride mail a myslim ze tych vyhod mat obsah kategorizovany v kategoriach a terminoch, je odost viac.

ten hook znie dobre, mas s tym nejake skusenosti? moja idea je, ze tam potom bude treba spravit aj to, aby sa dany taxonomy selectbox nezobrazoval pri vytvarani nodu. To je vlastne to, co mi vadi.

Pri dvoch „terminoch“

Napsal uživatel SonnY dne 17 Duben, 2008 - 16:04.

Pri dvoch „terminoch“ nema taxonomy vobec zmysel. Na toto je lepsie spravit si len dva typy obsahu s rovnakymi poliame a inym nazvom. Podla mna si to len zbytocne komplukujes ked silou mocou chces pouzit taxonomy.

Ak by si sa predsa len

Napsal uživatel mathiaz.sk@drup... dne 17 Duben, 2008 - 22:13.

Ak by si sa predsa len rozhodol pre tie terminy a ide ti iba o ukrytie selectboxu pre uzivatela mozno by bolo dobre vyuzit kombinaciu modulu taxonomy default a potom vlastneho mini modulu, co ukryje taxonomy box.

custom_forms.module

/**  * Implementation of hook_form_alter()  */ function custom_forms_form_alter($form_id, &$form) {   $content_types = array(     'hladam_spolujazdu_node_form'   );    if(in_array($form_id, $content_types)) {     $form['taxonomy']['#prefix'] = '<div style="display:none;">';     $form['taxonomy']['#suffix'] = '</div>';   } }

Ale upozornujem, ze toto riesenie iba skryje cast formulara, nie je to bezpecne riesenie aby ti tam niekto nenahodil inu hodnotu. Stale je mozne zobrazit tu cast formulara pomocou nastrojov ako je Firebug. Pokial by si to chcel uplne prednastavit nejaku hodnotu tak presny kod neviem, ale asi by som skusal zmenit typ form api konkretneho slovniku (vocabulary_id – vid) z select na value. Nasledovny kod je len na demonstraciu nemam ho odskusany a pisem ho priamo do textarea takze mozu byt chyby ;-)

/**  * Implementation of hook_form_alter()  */ function custom_forms_form_alter($form_id, &$form) {   // dodatocna struktura predstavuje cislo slovnika (vid), pre ktore   // by si chcel zmenit hodnotu a potom tiez cislo id terminu (tid),   // ktore chces mat prednastavene   $content_types = array(     'hladam_spolujazdu_node_form' => array(       'vid' => 10,       'tid' => 5     ),     'ponukam_spolujazdu_node_form' => array(       'vid' => 10,       'tid' => 6     ),   );    if(in_array($form_id, array_keys($content_types))) {     // aby sa ti nezobrazoval fieldset     $form['taxonomy']['#prefix'] = '<div style="display:none;">';     $form['taxonomy']['#suffix'] = '</div>';      // nahodia sa vlastne hodnoty, mozno troska bude treba pozmenit kod     // treba si vypisat formular pomocou print_r     // print_r($form['taxonomy']);     $key = $form['type']['#value'] .'_node_form';     $vid = $content_types[$key]['vid'];     $tid = $content_types[$key]['tid'];     $form['taxonomy'][$vid]['#type'] = 'value';     $form['taxonomy'][$vid]['#value'] = $tid;   } }

Hawk! :)
